WIP - OpenWRT install on ASUS RT-N12 VP

I recently got hold of two old ASUS routers. The routers are no longer in production, but in principle should be able to handle a few versions of OpenWRT (1) even though the hardware is "old" by current OpenWRT standards.

As the devices were a donation I feel a certain amount of liberty to explore, experiment and break things. A good learning project.

While my initial estimate assumed that it would take at most a week to:

  1. flash the OpenWRT OS onto the routers
  2. harden the OpenWRT installation
  3. automate the process to repeat on the second one

Three weeks later I am finaly done with step 1..... a good learning project indeed.

Problems and Solutions

USBGuard forces modeswtich on TPlink UE300

On certain machines this ethernet adapter will present itself as a mass storage device rather than as an ethernet adapter. I tried a number of solutions including using usb-modeswitch, editing modprobe.d and other voodoo without success. In the end it seems to be partly a conflict with USBGuard where the device after being blocked will go into an alternate state. Once in that state, for whatever reason I could not revert back to default. My working solution is to quickly allow the device after plugging it in (within seconds.)

TFTP Tips

These are old devices so the available troubleshooting information is a bit thin but in the end I got TFTP flash of the firmware working. The router runs a TFTP server allowing connections from devices with the 192.168.1.75 (some sites report 192.168.2.10) ip address when in recovery-mode. To be sure of successful upload, I suggest the following tftp settings:

  • binary
  • verbose
  • trace

I was able to flash the device with a variety of firmware files but outside of the stock ASUS firmware, the device would enter a boot loop (next section).

Kernel Panic

This was my biggest hurdle. After flashing the OpenWRT files, the device would enter into a boot loop. The device does not have a USB port but there is a UART interface. I removed the cover, soldered headers and using a FT232RL adapter was able to monitor the bootup of the device. At a certain point while loading the kernel the memory would run out.

To try to resolve this, I tried several solutions including:

  • Custom firmware builds - no success
  • Other firmware versions - no success
  • Builds for slightly different targets (MT7620) - no success

Until, for some reason I tried to just directly use the sysupgrade image of OpenWrt verion 21.02 and it worked..... I had tried using upgrade images earlier in the process and it did not work.

I dont know what changed. It worked after an unsuccesful flash for a slightly different target. Did that unset/clear a memory chunk? Did I need to do a certain number of flashes before the image took hold? Vodoo? Nonetheless, I now have a device running OpenWRT. Hardening, boot log snips and photos will be in an upcomming posts. And if, like me you are trying to get this working, I think the real trick might just be persistance....

Device Notes
